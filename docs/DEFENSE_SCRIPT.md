# 演讲答辩稿

各位老师好，我今天汇报的项目是《基于大模型的个性化资源生成与学习多智能体系统》。

本项目面向当前学习平台中普遍存在的几个问题：第一，学习资源往往是统一发放的，难以适配不同学生的基础水平和学习习惯；第二，学生练习、错题、学习报告之间的数据没有形成有效闭环；第三，单纯依赖大模型生成内容时，容易出现内容不稳定、缺少知识依据、无法结合真实学情等问题。

针对这些问题，我设计并实现了一个以大语言模型为核心、以多智能体协作为组织方式、以学习画像和知识库为支撑的个性化学习系统。系统能够根据学生的学习画像、历史作答记录、知识点掌握情况和教师划定的教学范围，自动生成个性化课件、习题、学习路径和反馈报告。

## 一、项目目标

本系统的核心目标可以概括为三点。

第一，构建动态学习画像。系统会根据学生的初始画像对话、答题正确率、作答耗时、薄弱题型和错题记录，持续更新学生的知识掌握度和学习偏好。

第二，生成个性化学习资源。系统不是简单生成一份通用课件，而是结合学生当前能力、知识库检索结果和知识点依赖关系，生成更适合当前学生的课件、习题和学习路径。

第三，形成学习反馈闭环。学生完成练习后，评价服务会更新作答记录、错题信息和掌握度，后续系统再次生成资源时，会使用最新画像，实现“学习、练习、评价、再推荐”的循环。

## 二、系统技术架构

在技术架构上，系统采用前后端分离和微服务设计。

前端使用 Vue 3 和 Element Plus，主要包括学生仪表盘、学习路径、课件生成、习题练习、错题本、知识图谱、智能问答、学习报告和教师工作台等页面。

后端主要使用 Python FastAPI 构建多个服务，包括用户服务、智能体服务、资源服务、评价服务、教师服务和系统服务。其中，智能体服务是系统的核心，负责多智能体编排、课件生成、习题生成、智能问答、学习路径规划、知识库检索和知识图谱查询。

在数据层，系统采用多库协同设计。PostgreSQL 用于保存用户、画像、资源、习题、作答记录和学习报告等结构化业务数据；Neo4j 用于保存知识点之间的前置依赖、后续推荐和资源关联关系；ChromaDB 用于保存知识库文档向量，支撑 RAG 检索；Redis 用于缓存；RabbitMQ 用于异步任务和事件传递；MinIO 用于保存课件等文件资源。

这种架构的好处是，不同类型的数据由最合适的存储系统处理，既能保证业务数据的稳定性，也能支持知识图谱推理和向量检索。

## 三、多智能体协作设计

本项目的一个重点是多智能体协作。系统中设计了多个专业智能体，包括学习画像智能体、知识图谱智能体、知识库智能体、路径规划智能体、资源生成智能体、习题生成智能体、答疑智能体和评价反馈智能体。

这些智能体通过 LangGraph 进行编排。用户发起请求后，协调器会先分析请求意图，再选择需要参与的智能体。例如，当学生请求生成某个知识点的课件时，系统会先调用学习画像智能体分析学生当前水平，再调用知识库和 RAG 检索相关内容，最后由资源生成智能体生成课件，并经过审核后保存到数据库。

这种设计避免了单个大模型一次性完成所有任务带来的不稳定问题。每个智能体只负责相对明确的子任务，系统整体更容易扩展，也更容易排查问题。

## 四、核心功能实现

系统主要实现了以下功能。

第一，个性化学习画像。系统从学生基础信息、学习目标、作答记录和错题情况中提取画像特征，包括知识掌握度、学习风格、认知能力和学习习惯等。

第二，个性化课件生成。系统会先构建生成计划，确定知识点、课件风格、难度、目标字数和个性化提示，然后结合 ChromaDB 检索到的知识库内容，让大模型生成 Markdown 格式课件。

第三，习题生成与错题闭环。系统可以根据学生掌握度动态调整题目难度比例，并支持选择题、填空题、判断题、简答题和编程题等多种题型。学生作答后，评价服务会记录结果并更新画像。

第四，智能问答。系统支持结合知识库和学生学习上下文进行答疑。对于简单问题，系统可以优先使用知识库和小模型处理；对于复杂问题，再调用大模型进行深入分析。

第五，学习路径规划。系统会根据学生当前掌握度和教师教学范围，生成分阶段学习任务，包括课件学习、习题练习、错题复盘和后续拓展。

第六，教师工作台。教师可以划定教学范围、发布课件和查看学生学习情况，使系统既能体现个性化推荐，也能符合教师的教学计划。

## 五、AI 工具使用过程

在开发过程中，我使用 AI 工具辅助完成了需求拆解、架构设计、Prompt 编写、代码生成、文档整理和测试检查。

具体来说，前期我使用 AI 工具将项目需求拆分为多个服务模块，并比较了单体架构、微服务架构和异步任务架构的优缺点。中期，我使用 AI 工具辅助设计课件生成、习题生成、智能问答和反馈综合等 Prompt 模板。后期，我使用 AI 工具辅助整理接口文档、系统架构文档和答辩材料。

但需要说明的是，AI 工具在本项目中主要承担辅助角色。关键的系统设计、模块边界、数据结构、功能取舍和最终验证，仍然由人工完成。整个过程可以概括为“人工提出目标，AI 辅助生成，人工检查修正”。

同时，在系统运行阶段，大模型也是业务能力的一部分。它参与学习画像分析、课件生成、习题生成、智能答疑和反馈总结，但系统通过 RAG 检索、知识图谱约束和内容审核机制，尽量降低大模型幻觉，提高输出稳定性。

## 六、项目创新点

本项目的创新点主要有四个。

第一，多智能体协同生成学习资源。系统不是简单调用一次大模型，而是让画像、知识库、图谱、路径规划、资源生成和反馈评价等智能体分工协作。

第二，学习画像驱动的动态资源生成。不同学生即使学习同一个知识点，系统也会根据其掌握度、错题记录和学习偏好生成不同内容。

第三，RAG 与知识图谱结合。RAG 提供语义相关内容，知识图谱提供知识点依赖关系，两者共同增强大模型输出的依据性和结构性。

第四，形成完整学习闭环。系统将课件生成、习题练习、作答评价、错题分析、画像更新和再次推荐连接起来，使学习过程可以持续优化。

## 七、总结

总体来说，本项目构建了一个面向个性化学习场景的多智能体系统。它将大语言模型、RAG 检索、知识图谱、学习画像和微服务架构结合起来，实现了从资源生成到学习评价再到个性化推荐的完整流程。

通过这个项目，我进一步理解了大模型应用不能只停留在“生成文本”层面，而应该和真实业务数据、知识库、审核机制和反馈闭环结合起来，才能真正服务于教育场景。

我的汇报到此结束，感谢各位老师。
