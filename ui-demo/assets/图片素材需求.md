# 首页所需的图片/动图素材清单

## 概览

`landing.html` 中的 8 个功能模块卡片各留了一个图片占位区域。
以下是每个位置的建议素材说明，包括内容方向、建议格式和尺寸。

---

## 素材列表

### 1. 个性化学习路径 — 时间线展开动画
- **位置**：功能卡片 1/8
- **类型**：GIF / WebM 动图 / Lottie 动画
- **建议内容**：学习路径从空白到各阶段节点逐个出现的动画效果，模拟时间线向下展开，已完成阶段标记绿色勾
- **建议尺寸**：800×300px (2.67:1)
- **色彩方向**：与当前主题 accent 色协调

### 2. 智能课件生成 — 生成到展示流程
- **位置**：功能卡片 2/8
- **类型**：GIF / WebM 动图
- **建议内容**：左侧输入框输入主题 → 进度条加载 → 右侧弹出结构化课件（大纲 + 正文），展示 "RAG 检索中..." → "课件生成完毕" 的过程
- **建议尺寸**：800×300px
- **色彩方向**：科技感蓝绿渐变

### 3. 自适应练习测评 — 答题反馈流程
- **位置**：功能卡片 3/8
- **类型**：GIF / WebM 动图
- **建议内容**：用户点击选项 → 正确显示绿色对勾 + 解析弹出 / 错误显示红色叉号 + 正确答案高亮
- **建议尺寸**：800×300px
- **色彩方向**：绿/红对比

### 4. 智能错题本 — 错题重练到巩固
- **位置**：功能卡片 4/8
- **类型**：GIF / WebM 动图
- **建议内容**：错题列表 → 点击 "生成变式题" → 新题目出现 → 用户作答 → 正确通过 → 错题标记为 "已掌握"
- **建议尺寸**：800×300px

### 5. 知识图谱可视化 — 网络交互视图
- **位置**：功能卡片 5/8
- **类型**：高清静态截图 / PNG
- **建议内容**：vis-network 渲染的知识图谱，节点带颜色区分掌握度（绿=掌握/橙=薄弱/灰=未学），边表示依赖关系。最好有一个节点被选中高亮，弹出 tooltip
- **建议尺寸**：800×400px
- **备注**：可直接使用项目实际运行截图

### 6. 多智能体问答 — 对话分析展开
- **位置**：功能卡片 6/8
- **类型**：GIF / WebM 动图
- **建议内容**：用户输入问题 → AI 回复文字 + 下方展开结构化分析卡片（知识差距 / 建议 / 路线更新）
- **建议尺寸**：800×300px

### 7. 阶段学习报告 — 仪表盘概览
- **位置**：功能卡片 7/8
- **类型**：高清静态截图 / PNG
- **建议内容**：报告面板概览 —— 正确率趋势图、知识覆盖度雷达图、薄弱点热力图等多图表并排展示
- **建议尺寸**：800×400px
- **备注**：可参考项目 evaluation_service 实际输出进行设计

### 8. 多角色协同 — 教师端界面
- **位置**：功能卡片 8/8
- **类型**：高清静态截图 / PNG
- **建议内容**：教师端班级管理界面 —— 学生列表 + 掌握度概览 + 作业布置面板，展示多角色系统的差异化
- **建议尺寸**：800×400px
- **备注**：可直接使用 TeacherView 实际运行截图

---

## 通用建议

| 项目 | 建议 |
|---|---|
| 格式 | WebM (动图) + PNG (静态) 双份 |
| 动图时长 | 3–6 秒循环 |
| 帧率 | 24–30 fps |
| 背景 | 透明或与暗色主题协调的深色背景 |
| 文件大小 | 动图控制在 500KB 以内，静态图 200KB 以内 |
| 制作工具 | ScreenToGif（免费）、LICEcap、After Effects + Bodymovin（Lottie）|

## 放置方式

将素材文件放入 `ui-demo/assets/` 目录，然后在 `landing.html` 中找到对应的 `.img-placeholder` 替换为：

```html
<img src="assets/your-file.gif" alt="描述" style="width:100%;border-radius:12px" />
```
