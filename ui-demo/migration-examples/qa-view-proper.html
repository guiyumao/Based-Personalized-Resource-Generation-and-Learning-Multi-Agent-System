      ❌ 错误做法 vs ✅ 正确做法

      CoursewareView / MistakeNotebookView 的风格冲突就是「错误做法」的典型案例

      ❌ 错误：保留旧的全屏壳
      <!-- CoursewareView.vue 当前写法 -->
<template>
  <div class="dashboard-shell student-workspace-shell">
    <div class="aurora aurora-a"></div>
    <div class="aurora aurora-b"></div>
    <main>
      <!-- 课件内容 -->
    </main>
  </div>
</template>

<style scoped>
/* 自己又定义了一套深色背景 */
.dashboard-shell {
  background: radial-gradient(...);
  min-height: 100vh;
}
</style>
      ✅ 正确：去掉壳，用主题变量
      <!-- CoursewareView.vue 正确写法 -->
<template>
  <div class="courseware-page">
    <!-- 直接放课件内容，AppLayout 已提供背景 -->
    <header class="page-hero">
      <h1>{{ title }}</h1>
    </header>
    <div class="reader-layout">
      <aside class="outline-sidebar">...</aside>
      <article class="content-area">...</article>
    </div>
  </div>
</template>

<style scoped>
/* 使用主题变量，不做全屏覆盖 */
.courseware-page {
  max-width: 1100px;
}
.outline-sidebar {
  background: var(--panel);
  border: 1px solid var(--line);
  border-radius: 16px;
}
</style>

      📖 为什么 CoursewareView 和 MistakeNotebookView 风格不统一

      这两个文件没有被修改——它们仍然使用旧的全屏布局（dashboard-shell student-workspace-shell）， 自带深色背景、极光装饰、和硬编码颜色。

      当 AppLayout 把它们包在侧边栏+顶栏布局中时，它们的全屏 CSS 和新布局冲突，导致：

      • 内容区有自己的 min-height: 100vh，在布局内撑出多余的滚动条
      • 极光装饰（.aurora-a/b）在布局内定位异常
      • 硬编码深色（#08111f）不跟随主题切换
      • CSS 网格叠加层（::before）与 AppLayout 背景重叠

      修复方法：去掉 dashboard-shell / student-workspace-shell / .aurora， 把面板色、文字色、边框色全部换为 var(--bg) / var(--panel) / var(--line) / var(--text)。
